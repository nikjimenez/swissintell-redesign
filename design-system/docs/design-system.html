01 · Principles

How this system thinks

Four principles guide every decision in the Swissintell design system. They exist so the site reads as Swiss, institutional and credible — without being cold or bureaucratic.

Principle 01

Institutional, not corporate

Swissintell is a nonprofit with 18+ years of history. The visual tone is confident and serious, not trying to look like a startup or a bank. Generous whitespace, restrained color, strong typography.

Principle 02

Red earns its weight

Red is the brand accent — not a background color. It signals action (CTAs), brand (logo, labels) or key emphasis. If everything is red, nothing is. Used sparingly, it directs the eye.

Principle 03

Dark for authority, light for content

Navy sections (hero, members, footer) project institutional weight. Light sections carry editorial content (events, press, verticals). Alternation creates rhythm and makes both feel intentional.

Principle 04

Motion is a whisper

Transitions are fast (150–250ms) and use a single easing curve. No bouncing, no parallax, no carousels. Motion acknowledges interaction — it never performs.

✓ Do

Use the scale as-is. Large headings (3xl, 4xl) get tight letter-spacing (-0.03em). Eyebrow labels are always uppercase with wide tracking (0.14em).

✗ Don't

Don't create intermediate sizes. Don't use Calibri italics for body text. Don't mix Calibri with other typefaces beyond the documented system fallbacks.

06 · Motion

Duration & easing

Three durations, two easings. That's the entire motion system. Consistency matters more than variety.

--duration-fast
150ms
Button hover, link color changes, toggle states.
--duration-base
200ms
Card hover, nav underline, backgrounds, most transitions.
--duration-slow
250ms
Larger layout shifts (card elevations, modals opening).
--ease-out
cubic-bezier(0.2, 0.8, 0.2, 1)
Default for entering/appearing elements. Feels responsive.
--ease-in-out
cubic-bezier(0.4, 0, 0.2, 1)
State changes where start and end are both visible (hover reversals).
Rule
Avoid
No bouncing, spring, or elastic easings. No auto-scrolling carousels. No parallax.

✓ Do

Use one primary button per section. Pair it with a ghost if a secondary action exists. Keep labels concise (2–4 words).

✗ Don't

Don't place two primary buttons adjacent — it creates competing CTAs. Don't invent new button colors or sizes.

10 · Patterns

Layout patterns

How sections are composed. Every homepage section follows one of these three patterns.

Pattern A

Section header (content-centric)

Eyebrow label (red, uppercase) → h2 title → lead paragraph. Optional "view all" link aligned right. Used in Events, Verticals, Press Review.

Pattern B

Split section (50/50)

Text column + visual column, 50/50 on desktop, stacked on mobile. Used in Members Area (text + resource card) and Newsletter (text + form).

Pattern C

Hero + stats bar

Full-width dark hero section, followed by a dark stats bar (4 metrics, dividers between). Used once, at the top of the homepage only.

Rule

Rhythm by alternation

Dark → Light → White → Dark → Light → Dark. Never stack two dark or two white sections in a row. This alternation creates visual rhythm.

11 · Usage

How to use these files

Two CSS files. Import them in order. Build on top of the component layer — never overwrite tokens inline.

/* In your WordPress theme header or main stylesheet */
@import "tokens.css";    /* foundation variables */
@import "components.css"; /* reusable components */
✓ Do

Reference every color, size, space and shadow via its token (e.g. var(--space-8)). Extend the system by adding new tokens, not overriding values.

✗ Don't

Don't hardcode hex colors or pixel values in page templates. Don't duplicate components with small variations — consolidate into the system.
